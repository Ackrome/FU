# Введение

Моделирование физических процессов является одной из важнейших задач в современной вычислительной математике и программировании. Классическая задача N-тел (N-body problem), заключающаяся в моделировании гравитационного взаимодействия множества объектов, находит применение как в фундаментальной астрофизике, так и в аэрокосмической инженерии для расчета траекторий космических аппаратов.

Актуальность данного проекта обусловлена необходимостью создания наглядных, высокопроизводительных и интерактивных инструментов для изучения орбитальной механики. Большинство существующих аналитических решений применимы лишь для задачи двух тел, в то время как расчет сложных многокомпонентных систем требует применения методов численного интегрирования.

Целью данной работы является разработка программного комплекса на языке программирования C, предназначенного для симуляции гравитационного взаимодействия космических тел в реальном времени с поддержкой графического отображения. 

Для достижения поставленной цели были сформулированы следующие задачи:
1. Изучение математического аппарата (закон всемирного тяготения Ньютона, методы численного интегрирования).
2. Проектирование структур данных для хранения информации о физических объектах.
3. Разработка физического ядра (движка) для перерасчета координат, скоростей и ускорений тел.
4. Реализация системы обработки неупругих столкновений.
5. Создание подсистемы прогнозирования будущих траекторий (оракул Кеплера).
6. Разработка интерактивного графического интерфейса пользователя с использованием графической библиотеки Raylib.
7. Написание модуля автоматического модульного тестирования (Unit-тестирование) для верификации математических расчетов.

# 1 Постановка задачи и выбор инструментов

## 1.1 Требования к программному обеспечению

Программный продукт должен представлять собой оконное приложение, способное обрабатывать физику взаимодействия десятков тел с частотой обновления не менее 60 кадров в секунду (FPS). Программа должна позволять пользователю:

* Изменять масштаб отображения и перемещать камеру по виртуальному пространству.

* Изменять скорость течения времени (Time Warp) с использованием экспоненциального сглаживания.

* Создавать новые физические объекты с заданными параметрами массы и вектора скорости.

* Автоматически рассчитывать первую космическую скорость для вывода объекта на идеальную круговую орбиту относительно родительского тела.

## 1.2 Выбор средств разработки

Для отрисовки графики, рендеринга текста и обработки пользовательского ввода (клавиатура, мышь) используется кроссплатформенная графическая библиотека **Raylib**. Она предоставляет низкоуровневый доступ к OpenGL, что оптимально подходит для рендеринга тысяч точек (следов орбит) в реальном времени без существенного падения производительности.

Для сборки проекта применяется система **CMake** совместно с пакетным менеджером **vcpkg**, что обеспечивает переносимость исходного кода и упрощает управление зависимостями в средах MSVC (Microsoft Visual Studio).

# 2 Архитектура программы и описание структур данных

Архитектура разработанного приложения базируется на процедурной парадигме программирования. Состояние всей физической симуляции хранится в глобальных массивах данных, что позволяет избежать переполнения стека вызовов (Stack Overflow) при выделении памяти под крупные массивы прогнозирования орбит.

В основе физической модели лежит собственный тип данных для представления двумерных векторов двойной точности. 

```c
typedef struct { 
    double x; 
    double y; 
} Vec2D;
```

Использование типа `double` (64-битные числа с плавающей запятой) критически важно для масштабов Солнечной системы, где расстояния измеряются триллионами метров, а гравитационная постоянная имеет порядок $10^{-11}$. Тип `float` (32-бита) приводил бы к катастрофическим погрешностям округления.

Основной сущностью предметной области является структура `Body`, описывающая физическое тело (планету, звезду или космический аппарат).

```c
typedef struct {
    char name[32];         // Текстовое название объекта
    double mass;           // Масса тела в килограммах
    double radius;         // Физический радиус в метрах
    Color color;           // Цвет объекта для рендеринга
    Vec2D pos;             // Вектор текущей позиции в пространстве
    Vec2D vel;             // Вектор текущей скорости
    
    Vec2D trail[TRAIL_LEN]; // Кольцевой буфер для хранения прошлых позиций (хвост)
    int trailIdx;          // Текущий индекс кольцевого буфера
    Vec2D lastTrailPos;    // Координаты последней записанной точки хвоста (для сглаживания)
    
    Vec2D pred[PRED_LEN];  // Массив точек предсказанной будущей траектории
} Body;
```

Для хранения тел используется статический массив `system[MAX_BODIES]`. Также предусмотрен массив `sim[MAX_BODIES]`, который служит "теневой копией" системы и используется исключительно внутри алгоритма прогнозирования для независимого прогона времени в будущее без искажения основного состояния симуляции.


# 3 Описание программных модулей и функций

Программный комплекс разделен на логические блоки, реализованные в виде отдельных функций. В данном разделе приведено детальное описание 10 основных функций, составляющих ядро физического движка и системы рендеринга.

## 3.1 Модуль преобразования координат

В связи с астрономическими масштабами виртуального мира (миллиарды километров), прямое отображение координат на экран невозможно. Для этого разработаны функции проецирования между мировой (физической) и экранной (пиксельной) системами координат.

### 3.1.1 Функция WorldToScreen

**Назначение:** Преобразование мировых координат физического объекта в координаты пикселей на экране с учетом текущего положения камеры и масштаба.

**Входные аргументы:**

* `Vec2D worldPos` — мировые координаты объекта (метры);
* `int sw` — текущая ширина окна приложения (пиксели);
* `int sh` — текущая высота окна приложения (пиксели).

**Возвращаемое значение:** Тип `Vector2` (структура библиотеки Raylib), содержащий экранные координаты $X$ и $Y$.
**Логика работы:**

```c
Vector2 WorldToScreen(Vec2D worldPos, int sw, int sh) {
    return (Vector2){ 
        (float)((worldPos.x - camX) / scale + sw / 2.0), 
        (float)((worldPos.y - camY) / scale + sh / 2.0) 
    };
}
```

### 3.1.2 Функция ScreenToWorld

**Назначение:** Обратное преобразование координат курсора мыши в физические координаты виртуального мира (используется для создания новых объектов).

**Входные аргументы:**

* `Vector2 screenPos` — координаты пикселя на экране;
* `int sw`, `int sh` — ширина и высота окна.

**Возвращаемое значение:** Тип `Vec2D`, содержащий мировые координаты в метрах.
**Логика работы:**

Функция умножает смещение пикселя относительно центра экрана на глобальный коэффициент `scale` и прибавляет смещение камеры `camX`, `camY`.

## 3.2 Физический движок и математические расчеты

### 3.2.1 Функция StepPhysics

**Назначение:** Вычисление гравитационного воздействия между всеми телами в системе и обновление их позиций (численное интегрирование).

**Входные аргументы:**

* `Body* bodies` — указатель на массив космических тел;
* `int count` — текущее количество тел в системе;
* `double dt` — шаг интегрирования (дельта времени в секундах).

**Возвращаемое значение:** Отсутствует (`void`). Изменяет состояние переданного массива тел.

**Логика работы (вычисление сил):**

Реализован алгоритм сложности $O(N^2)$, базирующийся на законе всемирного тяготения Ньютона. Для предотвращения деления на ноль при сингулярностях введена проверка на минимальную дистанцию (сумму радиусов тел).

```c
double dx = bodies[j].pos.x - bodies[i].pos.x;
double dy = bodies[j].pos.y - bodies[i].pos.y;
double distSq = dx*dx + dy*dy;
double dist = sqrt(distSq);

if (dist < bodies[i].radius + bodies[j].radius) {
    dist = bodies[i].radius + bodies[j].radius;
}
double force = (G * bodies[j].mass) / distSq;
acc[i].x += force * (dx / dist);
acc[i].y += force * (dy / dist);
```

### 3.2.2 Функция HandleCollisions

**Назначение:** Обнаружение и обработка абсолютно неупругих столкновений (слияние объектов).

**Входные аргументы:**

* `Body* bodies` — указатель на массив тел;
* `int* count` — указатель на целочисленную переменную количества тел.

**Возвращаемое значение:** Отсутствует (`void`).

**Логика работы:**

При обнаружении коллизии (дистанция между центрами меньше суммы радиусов), вычисляется новая скорость объединенного тела на основе закона сохранения импульса: $\vec{v} = \frac{m_1\vec{v}_1 + m_2\vec{v}_2}{m_1+m_2}$. Массы суммируются, а новый радиус вычисляется через кубический корень суммы объемов.

```c
double totalMass = bodies[i].mass + bodies[j].mass;
bodies[i].vel.x = (bodies[i].vel.x * bodies[i].mass + bodies[j].vel.x * bodies[j].mass) / totalMass;
// ... (аналогично для осей y и позиций pos)
bodies[i].radius = pow(pow(bodies[i].radius, 3) + pow(bodies[j].radius, 3), 1.0/3.0);
bodies[i].mass = totalMass;
// Сдвиг массива для удаления поглощенного тела j
for (int k = j; k < *count - 1; k++) bodies[k] = bodies[k+1];
(*count)--;
```

### 3.2.3 Функция CalculateOrbitalVelocity

**Назначение:** Автоматический расчет вектора скорости для выхода на идеальную круговую орбиту вокруг заданного родительского тела (вычисление первой космической скорости).

**Входные аргументы:**

* `Body parent` — структура родительского тела (например, Солнца);
* `Vec2D targetPos` — координаты точки старта нового объекта.

**Возвращаемое значение:** Тип `Vec2D` (рассчитанный вектор скорости).

**Логика работы:**

```c
double v_mag = sqrt((G * parent.mass) / dist);
return (Vec2D){ 
    parent.vel.x + (dy / dist) * v_mag, 
    parent.vel.y - (dx / dist) * v_mag 
};
```

### 3.2.4 Функция CalculateBarycenter

**Назначение:** Нахождение центра масс (барицентра) системы тел.

**Входные аргументы:** `Body* bodies` (массив тел), `int count` (количество).

**Возвращаемое значение:** Тип `Vec2D` (мировые координаты барицентра).

## 3.3 Прогнозирование и интерфейс

### 3.3.1 Функция UpdatePredictions
**Назначение:** Расчет будущих траекторий движения для отрисовки линий прогноза (Оракул Кеплера).

**Входные аргументы:**

* `Body* real_bodies` — указатель на массив реальных тел;
* `int count` — количество тел;
* `double current_scale` — текущий масштаб отображения для адаптивной точности.

**Возвращаемое значение:** Отсутствует (`void`). Данные записываются в поля `pred` структуры тел.

**Логика работы:**

Функция копирует состояние вселенной в теневой массив `sim`, после чего запускает физическую симуляцию на несколько лет вперед в рамках одного кадра.

На этапе разработки была выявлена проблема дискретизации: при больших значениях ускорения времени (Time Warp) шаг интегрирования становился слишком велик, что приводило к отрисовке угловатых орбит в виде многоугольников (Рисунок 1). Проблема была решена внедрением адаптивного сглаживания и расчета шага на основе переменной `current_scale`.

## 3.4 Подсистема тестирования (Unit-тестирование)

### 3.4.1 Функция RunTests
**Назначение:** Автоматическая верификация математического и физического аппарата программы перед запуском графического окна. Выполняет требования надежности ПО.

**Входные аргументы:** Отсутствуют.

**Возвращаемое значение:** Отсутствует. В случае провала теста выполнение программы прерывается макросом `assert`.

**Логика работы:**

Искусственно создается изолированная система из двух тел, летящих навстречу друг другу. После вызова функции столкновений `HandleCollisions` проверяется выполнение закона сохранения импульса: если массы и скорости тел равны, итоговое тело должно полностью остановиться (`vel.x == 0.0`).
```c
Body b1 = {"A", 10.0, 1.0, WHITE, {0,0}, {5.0, 0}};
Body b2 = {"B", 10.0, 1.0, WHITE, {1.5, 0}, {-5.0, 0}}; 
Body testSystem[2] = {b1, b2};
int tCount = 2;
HandleCollisions(testSystem, &tCount);
assert(tCount == 1); 
assert(testSystem[0].vel.x == 0.0); 
```


# 4 Результаты работы и тестирование

## 4.1 Реализация подсистемы создания орбитальных спутников

В рамках выполнения требований к программному продукту была разработана специализированная функция интерактивного создания новых физических тел с автоматическим расчетом орбитальных параметров. 

При инициализации нового объекта (с помощью левой кнопки мыши) пользователь имеет возможность зажать клавишу `SHIFT`. Программный комплекс переходит в режим «Авто-орбиты». В этом режиме алгоритм определяет текущую цель (объект, на котором сфокусирована камера пользователя посредством переменной `focusIdx`). Если ни одна планета не выбрана, родительским телом по умолчанию назначается Солнце (индекс 0).

Используя функцию `CalculateOrbitalVelocity`, программа вычисляет точный вектор скорости (первую космическую скорость), необходимый для вывода нового объекта на идеальную круговую орбиту вокруг выбранного родительского тела. Вектор скорости родительского тела прибавляется к вектору скорости спутника, что позволяет запускать новые луны вокруг планет, которые сами находятся в движении по Солнечной системе (например, создание искусственного спутника Юпитера в реальном времени).

## 4.2 Тестирование программного комплекса

Тестирование программного продукта проводилось на двух уровнях:

1. **Автоматическое модульное тестирование (Unit-тесты):** Перед запуском графического контекста вызывается функция `RunTests`. Тесты успешно подтвердили корректность математической модели, в частности — строгое выполнение закона сохранения импульса при неупругих столкновениях макроскопических объектов.

2. **Функциональное тестирование:** В ходе ручного тестирования симуляции при максимальных значениях экспоненциального ускорения времени (Time Warp) была подтверждена стабильность работы сглаживания орбитальных следов. Проблем с переполнением памяти (Stack Overflow) не выявлено благодаря использованию статического выделения памяти для массивов структур тел.
